# 交互模块丰富与四合院展示改动说明

本文档用于帮助项目报告撰写人员理解近期除赵州桥论文校正之外的页面功能与交互改动。相关改动主要集中在斗拱模块、榫卯模块、四合院模块，以及桥梁抗震页面的布局优化。文档说明每项改动的目的、涉及文件、页面表现和可写入报告的表述方式。

## 一、改动目的

本轮调整的核心目标是把原本偏静态的建筑展示页面，进一步扩展为具有结构解释能力的交互式教学模块。调整后，用户不只是观看 3D 或 2D 模型，还可以通过滑块、按钮、分层高亮、拆装过程和环境模拟理解古建筑的结构逻辑。

具体目标包括：

- 让斗拱页面展示屋顶荷载如何从梁、枋、斗、拱逐层传递到柱子。
- 让榫卯页面从单一节点展示升级为可选择榫型、可拆装、可观察受力方向的实验台。
- 让四合院页面在保留现有内容的基础上，补充 2D/3D 展示切换、日照、通风和家庭等级空间解读。
- 让桥梁抗震页面减少大面积空白，使 3D 视口、控制面板和指标卡更紧凑。

## 二、涉及文件

| 文件 | 改动重点 |
| --- | --- |
| `src/pages/DouGongPageEnhanced.tsx` | 新增斗拱受力传递动画、荷载滑块、构件受力高亮和受力步骤说明 |
| `src/pages/SunmaoPage.tsx` | 新增榫型选择、拆装交互、插入深度控制和受压/受拉/抗剪说明 |
| `src/pages/SiheyuanPage.tsx` | 新增四合院 2D/3D 展示切换、可视化院落模型、日照轨迹、通风路径和家庭等级空间面板 |
| `src/pages/BridgePage.tsx` | 调整桥梁抗震页面布局，压缩空白，提高视口与控制区的信息密度 |

## 三、斗拱模块改动

### 1. 新增受力传递逻辑

斗拱模块新增了逐层受力传递演示。页面现在可以按照“屋顶荷载 - 梁枋层 - 拱昂层 - 斗层 - 柱础”的顺序进行高亮，让用户看到荷载如何从屋顶逐级传到柱子。

该改动把斗拱从单纯的 3D 构件展示，扩展为能够解释结构稳定性的教学模型。它重点说明斗拱并不是装饰构件，而是通过多层木构件的叠合、承压、出挑和摩擦约束实现荷载分散。

报告可参考表述：

> 斗拱模块新增“受力传递”可视化功能，将屋顶荷载从梁枋、斗、拱逐层传递到柱子的过程进行动态高亮展示。用户可以直观看到斗拱通过多级构件分散竖向荷载和水平扰动，从而理解传统木构建筑“无钉连接仍能稳定”的结构原因。

### 2. 新增荷载滑块和颜色反馈

页面新增了荷载大小控制。用户拖动荷载滑块后，斗拱构件的颜色会随受力程度变化，受力越明显的构件会以更醒目的颜色显示。

该设计将“荷载大小”从文字说明变成可操作变量，使用户能够观察同一斗拱体系在不同荷载输入下的响应差异。

报告可参考表述：

> 系统通过荷载滑块和构件颜色反馈，将斗拱结构中的力学变化转化为可交互的视觉信号。不同荷载水平下，构件颜色和高亮层级发生变化，帮助用户理解斗拱的分层承压与荷载扩散机制。

### 3. 新增功能说明面板

页面同步补充了“受力传递”和“新增功能”相关说明，用于提示当前演示关注的结构重点。该部分让页面从视觉展示过渡到结构知识讲解，便于报告中说明系统的教育价值。

## 四、榫卯模块改动

### 1. 新增榫型实验台

榫卯模块新增了多种榫型选择，包括直榫、燕尾榫、抱肩榫和穿带榫。每种榫型都有对应的简短说明、结构特征和颜色表现。

这使榫卯页面不再只展示一种通用连接，而是能够对比不同榫型的连接逻辑和力学侧重点。

报告可参考表述：

> 榫卯模块新增“榫型实验台”，支持直榫、燕尾榫、抱肩榫和穿带榫等典型结构的切换展示。系统通过不同几何形态说明榫卯连接在抗拔、限位、承压和拉结方面的差异，增强了古代木构连接技术的可解释性。

### 2. 新增拆装交互

页面新增榫头插入深度滑块，并提供“拆开、半插入、锁合”三个快捷状态。用户可以控制榫头从分离到插入再到锁合的过程。

该改动让用户理解榫卯连接的稳定性来自构件之间的几何咬合和接触面约束，而不是钉子或胶粘。

报告可参考表述：

> 榫卯模块新增拆装交互，用户可以拖动滑块观察榫头从分离、半插入到完全锁合的全过程。该功能将传统榫卯“可拆、可装、可锁固”的结构特征转化为可操作的实验过程。

### 3. 新增三向受力说明

每种榫型新增受压、受拉和抗剪方向说明。页面会根据当前选择的榫型展示不同的力学解释，例如燕尾榫强调几何抗拔，抱肩榫强调肩部限位，穿带榫强调贯穿拉结。

报告可参考表述：

> 系统为不同榫型补充了受压、受拉和抗剪三类受力说明，使榫卯节点从构造展示扩展为力学分析展示。用户能够理解不同榫型在不同受力方向下的优势和适用场景。

## 五、四合院模块改动

### 1. 保留当前页面并新增 2D/3D 展示切换

四合院页面新增了“2D展示”和“3D展示”选择按钮。默认展示当前可见的 2D 可视化内容，用户切换到 3D 展示后，会使用接近最初版本的四合院 3D 模型展示方式。

该改动解决了四合院页面需要同时保留当前内容和恢复原有 3D 模型展示的问题。2D 展示适合结构关系讲解，3D 展示适合空间形态观察。

报告可参考表述：

> 四合院模块新增 2D/3D 展示切换机制。2D 模式用于清晰呈现正房、厢房、倒座、院门、影壁和院心之间的空间关系；3D 模式则保留原有模型浏览体验，支持用户从立体视角观察院落形态。

### 2. 新增院落空间可视化模型

页面新增了四合院平面化可视模型，用 CSS/HTML 方式呈现正房、东西厢房、倒座、院门、影壁和院心等空间要素。该模型不依赖 WebGL 渲染，因此能够稳定展示四合院的基本空间组织。

报告可参考表述：

> 四合院页面新增稳定的 2D 院落可视化模型，用于表现四合院“中轴对称、南北进深、东西围合”的空间组织。该模型把建筑构成关系直接呈现在界面中，便于用户理解传统院落的平面秩序。

### 3. 新增日照轨迹演示

四合院模块新增季节和时间选择，包括春、夏、秋、冬，以及清晨、正午、傍晚等时间状态。用户可以观察不同时间与季节下院落光照和阴影关系的变化。

报告可参考表述：

> 四合院模块新增日照轨迹演示，通过季节和时段切换表现院落空间的光照变化。该功能用于说明四合院在朝向、围合高度和院落尺度上对采光与居住舒适性的影响。

### 4. 新增通风路径演示

页面新增夏季南风和冬季北风两类通风状态，用于展示院门、院心和厢房之间的空气流动关系。该功能强调四合院通过围合与开口组织自然通风和避风。

报告可参考表述：

> 四合院模块新增通风路径演示，通过气流线表现夏季通风和冬季避风的空间逻辑。用户可以观察院门、院心与房屋围合对空气流动的引导作用，从环境性能角度理解传统院落设计。

### 5. 新增家庭等级空间解读

页面新增家庭等级空间面板，用户可以选择正房、厢房、倒座和院门等空间区域，查看其居住角色和礼制含义。该功能把四合院的空间布局与传统家庭秩序联系起来。

报告可参考表述：

> 四合院模块新增家庭等级空间解读，将正房、厢房、倒座和院门等区域与居住身份、礼制秩序和家庭结构相对应。该功能使四合院展示不只停留在建筑形态层面，也能解释其社会文化内涵。

## 六、桥梁抗震页面布局改动

桥梁抗震页面在保持内容不变的前提下进行了布局优化。主要调整包括压缩标题区和卡片间距、扩大 3D 视口比例、将右侧控制面板改为更紧凑的栅格布局、缩短图表高度，并让论文校正要点和指标卡更靠近主要操作区。

该调整的目标是减少页面空白，使用户在一个屏幕范围内看到更多核心内容，提升桥梁抗震模块的整体协调性。

报告可参考表述：

> 桥梁抗震页面完成布局优化，在不改变原有教学内容的前提下压缩空白区域，提升 3D 视口、地震控制、荷载控制、应力图表和结构说明之间的信息密度。优化后的页面更适合课堂演示和项目答辩展示。

## 七、整体报告建议写法

可以将本轮改动概括为：

> 项目对多个古建筑交互模块进行了结构化升级。斗拱模块新增受力传递动画和荷载颜色反馈，榫卯模块新增榫型选择、拆装交互和三向受力说明，四合院模块新增 2D/3D 展示切换、院落空间可视化、日照轨迹、通风路径和家庭等级空间解读，桥梁抗震页面则在保持内容不变的基础上优化布局、减少空白。上述改动使系统从静态建筑展示进一步转向“可操作、可观察、可解释”的数字化教学平台，提升了项目在结构科普、文化阐释和交互体验方面的完整度。

## 八、改动价值

本轮改动的价值主要体现在四个方面：

1. 提高交互性：用户可以通过滑块、按钮和模式切换主动探索建筑结构。
2. 提高解释力：页面从模型展示扩展到受力路径、拆装逻辑、环境性能和空间礼制解释。
3. 提高展示稳定性：四合院新增 2D 可视模型，保证在 3D 模型不可见时仍能清晰传达空间关系。
4. 提高答辩表达效果：各模块新增的控制面板和说明卡片能够直接支撑项目报告与现场演示。
