# 项目影响因子算法详解

## 一、概述

在本项目中，“影响因子”并不是学术论文中的期刊影响因子，而是项目针对中国古建筑建立的一个**综合影响等级指标**。它用于描述某一建筑在中国建筑史中的代表性、历史地位、文化传播度和技术价值。

从实现方式来看，项目中的影响因子采用的是：

- **人工标注的离散评分**
- **规则驱动的可解释算法**
- **用于可视化与推荐系统的核心特征字段**

也就是说，影响因子本身不是运行时自动学习出来的结果，而是先由项目数据层给出，再由程序把它转化为可视化表达和推荐计算依据。

---

## 二、影响因子的字段定义

在项目的建筑数据结构中，影响因子字段定义如下：

```ts
interface Building {
  id: number;
  name: string;
  year: number;
  dynasty: string;
  desc: string;
  tech: string;
  category: string;
  impactFactor: number;
  features: TechFeatures;
}
```

其中：

- `impactFactor` 表示建筑影响因子
- 数据类型为 `number`
- 在当前项目中主要采用 `1 ~ 10` 的评分区间

该字段存在于建筑演化星图的数据源中，也被智能荐览模块复用。

---

## 三、影响因子的来源与本质

### 1. 不是自动训练得出的结果

从代码实现可以看出，影响因子不是通过机器学习模型训练得到，也不是运行时根据公式从原始数据自动计算得出，而是直接写入建筑数据，例如：

- 北京故宫太和殿：`impactFactor: 10`
- 洛阳白马寺：`impactFactor: 10`
- 应县木塔：`impactFactor: 10`
- 部分区域性、代表性稍弱的建筑可能为 `5`、`6`、`7`

因此，项目中的影响因子本质上是：

**基于建筑史知识和项目策展逻辑的人为赋值结果。**

### 2. 影响因子的实际含义

虽然代码中没有写出一份形式化的评分表，但结合数据分布与使用场景，可以将影响因子的内涵概括为以下几个方面的综合评价：

- **历史地位**
  - 是否属于某类建筑中的奠基作、代表作、巅峰作

- **文化传播度**
  - 是否具有广泛知名度和文化认知度

- **建筑技术价值**
  - 是否体现重要的结构创新、营造技术或工艺特征

- **时代代表性**
  - 是否能代表某一朝代、某一类型建筑的发展水平

- **遗产与学术价值**
  - 是否具备重要的文保、考古、建筑史研究价值

因此可以将其理解为：

**影响因子 = 建筑历史地位 + 技术代表性 + 文化传播力 + 建筑史价值 的综合等级评分**

---

## 四、影响因子在建筑演化星图中的算法作用

影响因子在建筑演化模块中是一个非常关键的控制变量，它直接决定了节点在三维星图中的空间层级和视觉表现。

### 1. 用于建筑排序

在星图布局生成函数中，系统会先按照影响因子从高到低排序：

```ts
const sortedBuildings = [...buildings].sort((a, b) => b.impactFactor - a.impactFactor);
```

这意味着：

- 影响因子更高的建筑优先参与布局
- 高影响建筑在空间叙事中被优先放置在更核心位置

这一设计有助于形成“核心建筑—外围建筑”的结构层级。

---

### 2. 用于计算星图半径

系统使用以下公式把影响因子映射为星图中的目标半径：

```ts
const impactRatio = (10 - building.impactFactor) / 10;
const targetRadius = minRadius + impactRatio * (maxRadius - minRadius);
```

其中：

- `minRadius = 8`
- `maxRadius = 250`

### 公式解释

- 当 `impactFactor` 越高时，`impactRatio` 越小
- 当 `impactRatio` 越小时，`targetRadius` 越接近最小半径
- 因此高影响建筑会更靠近中心，低影响建筑会分布在外围

### 举例说明

#### 情况一：影响因子为 10

```ts
impactRatio = (10 - 10) / 10 = 0
```

则：

```ts
targetRadius = minRadius
```

即建筑位于星图核心区域。

#### 情况二：影响因子为 5

```ts
impactRatio = (10 - 5) / 10 = 0.5
```

则目标半径位于中间层。

#### 情况三：影响因子为 1

```ts
impactRatio = (10 - 1) / 10 = 0.9
```

则建筑会更接近外围区域。

### 设计意义

该算法把“历史影响力”转换成“空间距离”，实现了以下视觉语义：

- **中心位置 = 建筑史中的核心节点**
- **外围位置 = 历史影响相对较低或区域性更强的节点**

这种处理方式非常适合答辩展示，因为它让抽象的“重要程度”变成了可直接观察的空间层级。

---

### 3. 用于节点颜色映射

项目通过分段函数把影响因子映射成不同颜色：

```ts
function getImpactColor(factor: number): string {
  if (factor >= 10) return '#00D9FF';
  if (factor >= 9) return '#4488FF';
  if (factor >= 8) return '#9966FF';
  if (factor >= 7) return '#FF66CC';
  if (factor >= 6) return '#FFCC00';
  if (factor >= 5) return '#FF8833';
  if (factor >= 3) return '#FF4444';
  return '#888888';
}
```

### 颜色分级逻辑

- `10`：亮青色
- `9`：蓝色
- `8`：紫色
- `7`：粉色
- `6`：金黄色
- `5`：橙色
- `3~4`：红色
- `0~2`：灰色

### 算法特点

这是一种**分段离散映射**，而不是连续渐变。它的优点是：

- 层级区分明确
- 用户可以快速理解不同等级
- 图例表达清晰，适合答辩讲解

---

### 4. 用于节点大小映射

系统通过线性公式控制星点尺寸：

```ts
function getNodeSize(factor: number): number {
  return 0.8 + factor * 0.15;
}
```

### 含义解释

- 每个节点至少具有基础尺寸 `0.8`
- 每增加 1 分影响因子，尺寸增加 `0.15`

### 举例

- `factor = 10` 时，节点大小约为 `2.3`
- `factor = 8` 时，节点大小约为 `2.0`
- `factor = 5` 时，节点大小约为 `1.55`

### 设计意义

这样处理后：

- 高影响建筑更大
- 更容易被识别
- 与“靠近中心、颜色更亮”的视觉特征形成统一表达

即高影响节点在视觉上会被三重强化：

- 更靠近中心
- 更显眼的颜色
- 更大的尺寸

---

### 5. 用于星点发光与点击热区大小

在节点三维对象构建时，项目还会根据影响因子计算基础尺寸：

```ts
const baseSize = (0.15 + (node.impactFactor / 10) * 0.25) * targetScale;
```

这个 `baseSize` 会进一步影响：

- 核心球体大小
- 发光层大小
- 点击命中范围大小

因此，高影响因子建筑不仅看起来更像“主星”，而且在交互体验上也更容易被用户注意和点击。

## 五、影响因子在智能荐览算法中的作用

影响因子不仅用于星图展示，也被纳入建筑相似度推荐模型中。

对应代码如下：

```ts
const impactGap = Math.abs(baseBuilding.impactFactor - building.impactFactor);
const impactScore = Math.max(0, 8 - impactGap * 1.6);
score += impactScore;
if (impactScore >= 5) {
  reasons.push('历史影响力接近');
}
```

### 1. 数学表达式

设两座建筑的影响因子分别为 `f1` 和 `f2`，则：

```text
impactGap = |f1 - f2|
impactScore = max(0, 8 - 1.6 × impactGap)
```

### 2. 算法含义

这是一个**线性衰减评分模型**：

- 两座建筑影响因子越接近，得分越高
- 影响因子差距越大，得分越低
- 该项得分最高为 `8`
- 当差值足够大时，该项得分归零

### 3. 例子分析

#### 例一：完全相同

如果两座建筑影响因子一致：

```text
impactGap = 0
impactScore = 8
```

此时得到该项满分。

#### 例二：差 1 分

```text
impactGap = 1
impactScore = 8 - 1.6 = 6.4
```

说明它们在历史影响层面仍然很接近。

#### 例三：差 2 分

```text
impactGap = 2
impactScore = 8 - 3.2 = 4.8
```

仍有一定相似性，但明显下降。

#### 例四：差 5 分

```text
impactGap = 5
impactScore = 8 - 8 = 0
```

说明两者历史影响等级差异过大，不再为推荐结果贡献额外分值。

### 4. 设计意义

该项得分体现的是：

**推荐系统不仅比较建筑形式和结构特征，还会考虑两座建筑在历史层级上的相近性。**

这样可以避免出现“结构上相似，但历史分量差异极大”的建筑被错误优先推荐的问题。

---

## 六、影响因子在整体推荐模型中的权重位置

项目的建筑相似度推荐并不是只看影响因子，而是多维度综合评分。当前评分项包括：

- 同类别建筑：`+30`
- 结构类型相同：`+16`
- 屋顶形制相同：`+16`
- 同朝代或年代接近：最多 `+12`
- 材料体系相近：最多 `+12`
- 营造技术相近：最多 `+12`
- 影响因子接近：最多 `+8`
- 斗拱特征一致：`+6`

因此，影响因子在整个推荐模型中的地位可以理解为：

- 不是最高权重
- 不是唯一决定因素
- 但它是一个重要的“历史层级校正项”

也就是说，它让推荐结果不仅“形似”，而且“地位相近”。

---

## 七、为什么项目采用这种影响因子算法

### 1. 可解释性强

相比黑盒模型，这种影响因子设计非常适合答辩和教学展示。因为您可以清楚说明：

- 分值代表什么
- 分值如何影响可视化
- 分值如何参与推荐

### 2. 与文化项目特性匹配

古建筑项目不像电商推荐那样拥有大量用户行为数据，因此直接采用可解释的规则型建模更合理。

### 3. 有利于知识策展

项目本质上不仅是技术系统，也是一个数字策展系统。影响因子使策展者可以主动表达：

- 哪些建筑更核心
- 哪些建筑更具代表性
- 哪些建筑应该在展示中占据更重要位置

---

## 八、该算法的优点与局限

### 1. 优点

- **可解释性强**
  - 每一步都可以清楚讲明白

- **实现简单稳定**
  - 不依赖复杂模型训练和外部数据集

- **视觉效果突出**
  - 非常适合三维星图这种分层展示场景

- **推荐逻辑自然**
  - 能把历史地位纳入推荐体系

### 2. 局限

- **初始评分带有人为主观性**
  - 影响因子当前依赖人工知识赋值

- **缺少自动学习能力**
  - 无法根据用户行为自动更新评分

- **难以表达更细粒度差异**
  - 目前是整数或准离散评分，表达精度有限

因此，这套算法更适合：

- 展示型项目
- 教学型项目
- 文化可视化项目

而不属于强数据驱动的预测型算法。

## 九、答辩时可直接使用的表述

### 版本一：简洁版

本项目中的影响因子是一个 1 到 10 的建筑综合影响等级指标，用于表达古建筑在建筑史中的历史地位、文化传播力和技术代表性。系统将该分值映射到建筑演化星图的节点位置、颜色和大小中，同时在智能荐览中通过影响因子差值的线性衰减函数参与相似度评分，从而实现建筑历史影响力的可视化表达和可计算建模。

### 版本二：技术版

本项目采用可解释的规则驱动建模方式，将古建筑的综合影响力抽象为 `impactFactor` 字段。数据层先完成专家化赋值，运行层再通过三类映射实现其算法价值：第一，将影响因子映射为星图半径，形成中心—边缘的空间层级；第二，将其映射为颜色和尺寸，形成视觉显著性差异；第三，在推荐算法中引入 `impactScore = max(0, 8 - 1.6 × impactGap)` 线性衰减模型，使推荐结果兼顾结构相似与历史层级相近。

## 十、总结

项目中的影响因子算法，本质上是一套“**人工知识赋值 + 规则化计算使用**”的综合机制。

其逻辑可以概括为：

- **在数据层**，影响因子用于表达建筑的综合历史影响等级
- **在可视化层**，影响因子控制节点的空间位置、颜色、大小和视觉存在感
- **在推荐层**，影响因子通过差值衰减模型参与建筑相似度计算

因此，这套算法虽然不属于机器学习意义上的自动评分模型，但它非常适合本项目这种强调文化表达、知识组织和可解释展示的古建筑数字化场景。

最终，它实现的是：

**把“建筑史中的重要性”转化为“系统中可视、可算、可交互的数字特征”。**
